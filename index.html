语法

区分大小写

标识符

标识符，指变量、函数、属性的名字，或者函数的参数。

标识符规则：标识符第一个字符必须是字母、下划线或美元符号（$）,其他字符必须是字母、下划线、没有符号和数字。

按照惯例，ECMAScript标识符采用驼峰大小写格式

不能把关键字、保留字、true、false和 null用作标识符。

保留字可能在将来被用作关键字，也不能用作标识符

Number 类型

Number类型 使用IEEE754格式来表示整数和浮点数值
最基本的数值字面量格式是十进制，除十进制外，整数还可以通过八进制或者十六进制的字面量值来表示，在进行算术计算时，所有八进制 和 十六进制表示的数值最终都将被转换为十进制数值

浮点数值
该数值中必须包含一个小数点，并且小数点后必须至少有一位数字
由于保存浮点数值需要的内存空间是保存整数数值的两倍，因此ECMAScript会不失时机地将浮点数值转换为整数值

对于极大或极小的数值，可以使用科学计数法（e）表示法

数值范围
ECMAScript并不能保存所有的数值，ECMAScript能够表示的最小数值保存在Number.MIN_VALUE,最大值保存在Number.MAX_VALUE,如果计算结果超出了JavaScript数值范围，那么这个数值被自动转换为 Infinity值，如果这个数值是负数 则会被转换为 -Infinity

想要确定一个数值是否在最大最小数值直接，可以使用isFinite()函数，返回true就是在范围之内

NaN
非数值（Not a Number）是一个特殊的数值，这个数值用于表示一个本来要返回数组的操作数未返回数值的情况

任何数值除以0会返回NaN

NaN 本身有两个特点：
1.任何涉及NaN的操作都会返回 NaN
2.NaN与任何值都不相等，包括NaN本身

isNaN()函数接收一个参数，函数会帮我们确定这个参数 是否 “不是数值” 返回值为 true/false

isNaN()函数也适用于对象，对象调用isNaN()，会先调用对象的valueOf方法，然后确定该方法返回的值是否可以直接转换为数值，如果不能 在基于这个返回值再调用toString()方法，再测试返回值

数值转换
非数值转换为数值：Number()、parseInt()、parseFloat()

Number()可以用于任何数据类型，parseInt()、parseFloat() 用于把字符串转成数值。

Number()函数转换规则：
如果是Boolean值，true 和 false 将分别转为 1和 0
如果是数字值，只是简单的传入和返回
如果是null值，返回0
如果是undefined，返回NaN
如果是字符串:
1.如果只包含数字，则转为十进制数值
2.如果字符串中包含有效浮点格式，则将其转换为对应的浮点数值
3.如果字符串中包含有效的十六进制格式，则转换为相同大小的十进制整数值
4.空字符串转为0
5.如果字符串中包含除上述之外的字符则将其转换为NaN
6.如果是对象，则调用对象的valueOf()方法然后依照前面的规则转换返回值，如果转换结果是NaN,则调用对象的toString()方法，然后再依次按照前面的规则转换返回的字符串值

Number()函数转换字符串时比较复杂且不够合理，因此在处理整数的时候更长用的是parseInt()

3.5操作符

发表于 2020-03-08 | 分类于 JavaScript |

ECMAScript 操作符适用于很多值，字符串、数字值、布尔值、对象（调用valueOf和toString 方法，得到科操作的值）

一元操作符

只能操作一个值的操作符叫做一元操作符

1.递增和递减操作符
各有两个版本，前置型和后置型
前置型 操作符位于操作的变量之前，后置型则位于操作的变量之后。

执行前置递增和递减时，变量的值都是在语句被求值以前改变的

执行后置递增和递减 操作，是在包含他们的语句被求值之后才执行的

如果递增 递减是这条语句的唯一操作，那么前置或者后置并不会影响结果

undefined

前置递增递减、后置递增递减 这四个操作符对任何值都适用

2.一元加和减操作符

这两个操作符分别表示正数、负数

一元加操作符
以一个加号表示，放在数值前面，不会对数值产生任何影响

非数值应用一元加操作符时，像Number()转型函数一样对值进行转换，而对象是先调用valueOf 和（或）toString 方法，再转换得到的值

一元减操作符
主要用于表示负数，例如 将1 转换成 -1

非数值应用时，一元减操作符遵循与一元加操作符相同的规则

布尔操作符

布尔操作符一共有三个：非（NOT）、与（AND）、或 （OR）

逻辑非
逻辑非操作符由一个叹号表示（!）,无论什么数据类型的值，都会返回一个布尔值

模拟Boolean()转型函数的行为

逻辑与

逻辑与操作符由两个和号（&&）表示，可以应用于任何类型的操作数。

逻辑与操作属于短路操作，如果第一个操作数能够决定结果，那么就不会再对第二个操作数求值。

在有一个操作数不是布尔值的情况下，逻辑与操作就不一定返回布尔值

两个操作数都是布尔值时：两个操作数都为true时结果为true，其他情况都为false

遵循规则
1.如果有一个操作数是null,则返回null
2.如果有一个操作数是NaN,则返回NaN
3.如果有一个操作数是undefined，则返回undefined
4.如果第一个操作数是对象，则返回第二个操作数
5.如果第二个操作数是对象，则只有第一个操作数的求值为true的情况下才会返回该对象
6.如果两个操作数都是对象，则返回第二个操作数

逻辑或
逻辑或操作符由两个竖线（||）表示，可以应用于任何类型的操作数

逻辑或操作符也是短路操作符，也就是第一个操作数求值结果为true 则不会再对第二个操作数求值了

逻辑或和逻辑与类似，如果有一个操作数不是布尔值，逻辑或也不一定返回布尔值

两个操作数都是布尔值时：两个操作数有一个为true则返回true

遵循规则
1.如果两个操作数都是null,则返回null
2.如果两个操作数都是NaN,则返回NaN
3.如果两个操作数都是undefined，则返回undefined
4.如果第一个操作数是对象，则返回第一个操数
5.如果第一个操作数的求值结果为false，则返回第二个操作数
6.如果两个操作数都是对象，则返回第一个操作数

乘性操作符

乘性操作符定义了3个操作符：乘法、除法、求模
如果乘性计算的某个操作数不是数值，后台会先使用Number()转型函数将其转换为数值。

乘法
操作符由星号（*）表示
遵循规则
1.如果操作数都是数值，执行常规的乘法计算，如果乘积超出了ECMAScript数值的表示范围则返回 Infinity 或 -Infinity
2.如果一个操作数是NaN 则结果是NaN
3.Infinity 与 0 相乘 结果是NaN
4.Infinity 与非零 相乘 则结果是Infinity 或 -Infinity
5.Infinity与Infinity相乘结果是Infinity
6.如果一个操作数不是数值，则在后台调用Number()转为数值，再应用上面的规则

除法
操作符由斜线（/）表示

遵循规则
1.如果操作数都是数值，执行常规的除法计算，如果商操出了ECMAScript数值的表示范围则返回 Infinity 或 -Infinity
2.如果一个操作数是NaN则结果是NaN
3.如果是Infinity 被 Infinity 除，则结果是NaN
4.0被0除，结果是NaN
5.如果非零的有限数被零除则返回 Infinity 或 -Infinity
6.如果一个操作数不是数值，则在后台调用Number()转为数值，再应用上面的规则

求模
求模（余数）操作符由百分号（%）表示

遵循规则
1.如果操作数都是数值，执行常规的除法计算，返回除得的余数
2.如果被除数是无穷大，而除数有限大则结果是NaN
3.如果被除数有限大，而除数是零 则结果是NaN
4.如果是Infinity 被 Infinity 除，则结果是NaN
5.如果被除数有限大，而除数无限大则结果是被除数
6.如果被除数是0，则结果是0
6.如果一个操作数不是数值，则在后台调用Number()转为数值，再应用上面的规则

加性操作符

加性操作符包含加法、减法

加法

遵循规则
两个操作符都是数值
1.如果一个操作数是NaN 则结果是NaN
2.Infinity 加 Infinity 结果是 Infinity
3.-Infinity 加 -Infinity 结果是 -Infinity
4.Infinity 加 -Infinity 结果是 NaN
5.+0加+0，结果是+0
6.-0加-0，结果是-0
7.+0加-0，结果是+0
如果一个操作数是字符串
1.如果两个操作数都是字符串，则将第二个操作数与第一个操作数拼接起来
2.如果一个操作数是字符串，则另一个操作数转换为字符串，然后再讲两个字符串拼接起来

如果一个操作数是对象、数值或者布尔值
调用他们的toString()方法获得相应的字符串然后再应用前面的字符串规则，对于undefined和null,分别调用String()函数并获得字符串”undefined”和 “null”

如果期望结算完结果后在于字符串拼接

减法
减法操作符（-）

遵循规则
1.如果两个操作数都是数值，则执行常规的算术法操作并返回结果
2.如果一个操作数是NaN ，则结果是NaN
3.Infinity 减 Infinity 结果是NaN
4.-Infinity 减 -Infinity 结果是NaN
5.Infinity 减 -Infinity 结果是Infinity
6.-Infinity 减 Infinity 结果是-Infinity
7.+0减+0 结果是+0
8.+0减-0 结果是-0
9.-0减-0 结果是+0
10.如果一个操作数是字符串、布尔值 、null 、undefined 则先调用 Number()函数将其转换为数值，然后再根据前面的规则计算
11.如果有一个操作数是对象，则调用对象的valueOf()方法以获取表示该对象的数值，如果对象没有valueOf()方法，则调用其toString()方法并得到字符串转为数值

关系操作符

小于（<）、大于（>）、小于等于（<=）、大于等于（>=），比较贵重与数学一样，几个操作符都返回一个布尔值。

规则：
1.如果两个操作数都是数值，则执行数值比较
2.如果两个操作数都是字符串，则比较两个字符串对应的字符编码
3.如果一个操作数是数值，则将另一个操作数转换为一个数值，然后再执行比较
4.如果一个操作数是对象，则调用这个对象的valueOf()方法，用得到的结果按照前面的规则进行比较，如果没有valueOf方法，则调用toString方法，并用得到的结果根据前面的规则执行比较
5.如果一个操作数是布尔值，则将其转换为数值，然后再执行比较

函数在执行完return语句之后停止并立即退出，因此return 语句之后的任何代码都永远不会执行

一个函数可以包含多个return 语句，return 后不带任何返回值 将默认返回 undefined

推荐的做法，要么让函数始终返回一个值，要么永远都不要返回值，否则，如果函数有时候返回值，有时候不返回值，会给调试代码带来不便

理解参数

ECMAScript中的参数在内部是用一个数组来表示的，函数接收到的始终是一个数组，不关心数组中包含哪些参数。

函数体内可以通过arguments 对象来访问参数数组，arguments 对象只是与数组类似（不是Array的实例）

例子：

arguments的值 永远与对应命名参数的值保持同步

例子：

第二个参数的值修改为10，因为arguments对象中的值会自动反应到命名参数，所以修改arguments[1],也就修改了num2,它们的值都会变成10。但是它们的内存空间是独立的，只是值会同步

如果只传入了一个参数，那么为arguments[1]设置的值不会反映到命名参数中，这是因为arguments对象的长度是由传入的参数个数决定的。

没有传递值的命名参数将自动被赋予undefined值

严格模式下，重写arguments的值会导致语法错误

ECMAScript中的所有参数传递的都是值，不可能通过引用传递参数

没有重载

ECMAScript函数不能像传统意义上那样实现重载，在其他语言（如 Java）中，可以为一个函数编写两个定义，只要这两个定义的签名（接受的参数的类型和数量）不同即可。

ECMAScript 中定义了两个名字相同的函数，后定义的函数会覆盖先定义的

ECMAScript 可以通过检查传入函数中参数的类型和数量并作出不同的反应，可以模仿方法的重载。
